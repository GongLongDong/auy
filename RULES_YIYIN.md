## 1) 总原则（推理框架）

- **形参模式（多义绑定）**：一个单词可能对应多个含义（像编程形参）。分析时先选定“当前使用的是哪个含义/语境”，再回推字母/音素的正确解读；必要时允许**重设某个字母在该语境的解读**。
- **先体系后学术**：解释以本体系为基准，不用学术界默认框架来纠偏。
- **抽象/具象双轨**：
  - **抽象版**：注重量、状态转移、完成、封装、指向等结构性关系。
  - **具象版**：用该词的实际含义（生活/物理/操作场景）去描述过程变化。
  - 两轨允许同一音（如 `ei`）在不同轨道采用不同侧重。
- **覆盖/优先级机制**：同一音/字母可多义；在某些词形/语境下，某一义会覆盖另一义（类似 CSS 规则覆盖）。分析时要明确“本次主导义是什么、被覆盖义是否视为空”。
- **多义并用**：每个字母、每个音都可以挂**多个义位**；**一次解读允许同时选用多个义位**（不是强制只选一个），并在推理链里写明各自作用。
- **提出疑问的标准**：只在推理链条确实卡住时提问，问题必须指向“当前体系中已出现过的规则节点”，避免引入未触及的新问题域。
- **例句优先、话要说白**：新词或新义项**先给典型英文例句**（如 **A permits B to …**），再**顺着句子里谁对谁、准许的是什么事**往下拆；优先用**日常说法**，少堆术语。
- **抽象模式另章**：**量、节点、因果链**的压缩写法、与多词共用的「模板句」等，以后可单开一章集中写；本规则卡 **§6** 以**直观范例**为主，需要时再指「详见某章」而不重复铺全。

| **`li`（块）** | **`l` 连续输出** **`i` 的变化**；统一抽象下**「变化」**＝任意**行为或状态**变化（便于本质化）；**`l`** 强调**连续地输出**一串 **`i`** | **`list`**：**§6.40**；与 **`line`** 首段对读：**§6.39** |

| **`is`（块）** | **`i` 变化朝向**已锁定/已设的位（与 **`si` 顺序对翻**） | **`list` 入格** 等：**§6.40**、**§6.42** |
| **`st`（块）** | **`s` + `t`**：**锁定位 + 落定**；义理上与 **`set` 同脉**（**体系内等式**，**非历史词源**） | **`set`**：**§6.5**；**`list`**：**§6.40**；**§6.42** |

## 2.1) 字母承义规则（音 vs 字母位）

- **无音也可承义**：即使某字母在实际单词发音里不突出/弱化，也可能仍然有语义。
- **优先形义**：默认以**字母形义**作为承义主轴。形义来源于字母本音；通常接近字母表读音，但并不总相同（例：`a` 的本音不必等于字母名 `/ei/`）。
- **字母表音义**：字母名读音也可成为语义来源之一，但当前阶段默认优先级低于形义（除非在分析中被你显式启用）。
- **（待补）同位冲突**：同一字母位上，**字形义**与**实际读音变体**所支撑之义若不一致，**裁决顺序**（例如先绑 IPA 再选义位，或先字母位再校正音）**尚未写死**；弱读、异读词上推理易飘。**现阶段**：把 **字母 / 音** 的明确关系吃透；**系统性裁决**留待词例与继承树充实后再收（汇总见 **§9**）。

## 2.2) 辅音 + 元音：**产出模型**（方案 2，非学术界「修饰」套路）

> 本体系采用**方案 2**：**不用**「辅音修饰元音」这类**学术界常见表述**；改用 **「前段辅音（或辅音群）的产出 → 落在元音所表的量上」**。

- **辅音 + `e`（优先讨论）**：**几乎每个辅音**都可与 **`e` 拼合**，表示该辅音所代表的过程/动作的**输出结果**，该结果落在 **`e` 所表的「结束性变化 / 变化之量」**上。例：`**fe**`、`**he**`（**若有例外，由你单独声明**）。
- **辅音 + `a` / `o`（开始量）**：理论上也可表**开始量**（例：`**pa**`、`**fo**`）；但**开始量不止一种**，而**结束量在本体系中锚定在 `e`**，故**通常优先分析 `辅音 + e` 这一接点**。
- **`pa` 与 `ap`（输入 / 输出）**：**一对方向位**——**一个偏输出，一个偏输入**（你已用于 **`append`**：**`ap`** = **分离或分配 `A` 进入 `a`**，此时 **`a` 中含 `A`**；**`pa`** 走另一向，遇词再定）。**`ap` 侧**已有锚点；**`pa` 侧**尚缺 **§6** 级入库范例，遇 **`pa-`** 词时 **§2.5** 第 3～4 步易停**临时节点**（见 **§9**）。
- **辅音群 + `e`（示例）**：如 **`dep` + 后续 `e`**——不说「`dep` 修饰后面的 `e`」，而说：**`dep` 的产出 = 结束性变化 `e`**（**产出落在 `e` 上**）。
- **前缀块（已定标）**：**`im`** = **`i`（音 `/i/`）漫延** + **`m` 边界** → **漫延至边界**（见 **`impend`**，**§6.12**）。

## 2.3) 字母义与音义：**继承树 + 临时节点**

- **多义入库**：**每个字母**、**每个音**（含字母在词内的实际读音变体）都可以有**多个义位**；凡你已讲述过的，都应**记入继承树**（可与 **§3 `da→d`** 并列扩展为总树）。
- **父子未定**：若尚不清楚某义位从哪一父节点继承，先挂在 **「临时节点」** 下，**只记录义位与来源场景**，待后续统一整理父子关系。
- **字母 ≠ 音**：同一字母在不同读音下可对应不同音义分支；**字形义**与**音义**在树中**分别标注**，合并关系由你定（如 **`a` 合并模式**）。

## 2.4) `ec` / `ce`、`fa` / `fe`、`fact` / `fect`、质变与 **`e`**

- **`ec`**：**打破**（含**坍缩 `e`**）；**通常需与 `t` 组合**使用。
- **`ce`**：**攻克**——基底可关联**锁定目标**，衍生为**攻克**（攻克即锁定要拿下的事件/物）。
- **`fact` vs `fect`**：**`fact`** 强调 **量变 → 质变** 的积累；**`fect`** 焦点在 **质变点**。
- **`fa`**：**`a` 音 `/æ/`**（如 **`fact`**）→ **完全/全部**，**满足质变的量**；与 **`af`**（**一次、够触发的量**，**`a` 常 `/ə/`**）**方向/音位可对读**。
- **`fe`**：**直接落在质变化**；**`e` 音 `/ei/`** → **多次质变**；**`/ə/`** → **单次质变**。
- **单独 `e`**：多为**结束性变化量**；要强调 **已发生质变**，须经 **`fe`、`pe`** 等（**几乎所有辅音 + `e`** 皆可，**例外你补**）。
- **`fect` 内 `ct`**：**坍缩** → **最终质变**。
- **`perfect` 译「完美」**：诸质变在语用里**约定为正向**——**文化/俗成层**，**不单靠**音素链必然推出「善」。
- **（待补）质变 / 量变 / 「一次足够质变」**：全文多处使用（如 **`ad`/`af`**、**`fact`/`fect`**、**`-mit` 前条件**），但**操作定义**与**「量变到阈值」之分界**尚未写死；词例增多后再收紧，以免**不同前缀 + 同一词干**凭语感各写一套、难以自检一致（汇总见 **§9**）。

## 2.5) `a` + 辅音 / 辅音 + `a`：**推演簿**（遇词先用）

> **`ag` / `at` 等**随词补义位；**`as`/`sa`** 已定 → **§2.6**；**`as` 与 `ap`/`af` 等**辨向仍走下列顺序。

1. **辨向**：**`a`+辅音** 还是 **辅音+`a`**（如 **`af`↔`fa`、`ap`↔`pa`**）。
2. **写音**：该处 **`/æ/`、`/ə/`、`/ei/`…**（IPA）与义位绑定（例：**`fa`→`/æ/`**）。
3. **取辅音义**：只用表中**已入库**节点（**`f`/`p`/`d`/…**），**无则标临时节点**。
4. **拼后半**：**`mit`/`fect`/`pend`…** 等已定干，问**量从哪来、坍缩落谁**。
5. **推不动即停**，等你定标。

**已定标示例**：**`af`**、**`ap`**、**`ad`**、**`fa`（`/æ/`）**（见 **§2.2**、**§6.12**、**§6.16**、**§6.18**）。

## 2.6) 字母 `a` 的读音与「量」的性质；**`as`/`sa`**；**`st`（视程）**

- **`a` 读 `ɑ` 或 `æ`（词内主重读位）**：**不定数**——量**不钉死**，可漫向很大。
- **`a` 读 `eɪ`**：**固定量**——空间或数档**已锁界**，**非**「无穷场」（与 **`ɑ`/`æ` 不定数**对读）。
- **`a` 读 `ə`**：**单一整体 / 单一体 / 一个**（打包成一块），与 **§2** 表弱读 **`a`**、**`ad`/`af`** 等用法衔接。
- **`a` 读 `ɔ`（如在 *wall*）**：**围合空间**——**围栏、球内腔**等「**被圈住的一膛**」；**不必**数学意义上的**绝对闭合**。
- **`/æ/` 附加隐含义**（在「不定数」之上，**随形参启用**）：易关联**流体介质中的空间**（空气、水等）、**抽象场**（思想、观念等）。
- **`as`（字母 `a` + `s`，元音在后）**：**`a` 的量向外扩**至**可感知的边界**；**`s`**：**观察、感知**。在 **`waste`** 形参下：**外露**即**流失、耗掉**（**§6.29**），与 *vast* 的「扩到视程」**不同义项**。
- **`sa`（`s` + 元音 `a`）**：**锁定 `a` 那一档量**（与 **`as` 外扩**对读）。
- **通则**：**元音 + `s`** ≠ **`s` + 元音**；**`os`、`us` 等**具体义**随词再定**（词例仍少，**§9.1**）。
- **`st`（分析块，如 `vast`）**：**视线所能触及之处**（与 **§4** **`t`** 界位/状态并读）。
- **`s` 交叠**：同一词里 **`s`** 可同时参与 **`as`** 与 **`st`** 两截（**字母位交叠承义**）。

## 2.7) 布尔 **0 / 非零**；**`na`、`ne`、`ine`、`nad`；`ter`**

> 作**推理锚**：只看两档 **「零」** 与 **「非零」**；不必与形式科学里的布尔代数逐字等同。

- **`na`**：**朝 0 收敛**（减损、收束 **`a`**）；布尔读法下**量档收到零**。**`ina` 中 `na`**：**消耗的 `a` 来源在 `i`**。与 **项目终止**对读：**多段各自收到 0**（停工→撤具…）再**合成终 0**，**不必**理解成「从存在里抹掉」；**终极生命**等形参才近**开/关尽零**。
- **`ne`**：**就是 0**。**`n`**：**收敛**；**`e`**：**结束位**——收敛**停住**，**结果输出在零档**；源流 **`i`**。
- **`ine`**：**`i` 当前非 0** → 经 **`ne`** → **0**；**`e`** 承**结果（0）**；**不绕 `na` 长链**（**短路径 `ne`**，**非**「不经 `n`/`e`」的玄学直通）。
- **`nad` + `d`/`t`**：**`na`→0** 后由 **`d` 或 `t` 完成壳**把 **0 钉死**；**理论块**可表「**收到 0 且落定**」。**西班牙语 *nada***（什么都没有）作**义理锚**（**§6.38**；**非**历史词源证明）。
- **`ter`（块）**：**获取**待**终止或待裁定**的事务；**`t`/`r`** 并读（**§6.35**、**§6.36**）。

**何以 *not* 用 `o`**：**`n`/`t` 义理上皆可**参与「收到无」，***not*** 取 **`o`**：**字形近 `0`**，且 **`o` 表完全、到头**——**「完全的无」**；**`n` 收敛 + `o` 完全无 + `t` 落定**（**§6.37**）。

## 3) 音源场景、root 与含义继承树（以 `da` → `d` 为例）

> 含义继承树**以后可随讨论修正**；本节只收录你已明确讲述过的节点。**总树扩展、临时节点**的工作法见 **§2.3**。

### 音源场景与 root 型

- **音源场景**：含义来自**第一场景**（同一字母/音节可能有多个音源场景；**除非你再额外说明，否则只采用对话里已提到的那一种/几种**）。
- **root 型**：由该场景得到的基本义，多属 **root 层**；是否在具体分析中启用 root，由你提醒（未提醒时以词内形参与已约定节点为主）。

### `da`（root）与字母 `a` 的「合并模式」

- **拆解**：`da = d + a`（音层拆解）。
- **`da` 的第一音源场景**：下雨时，**水滴撞击地面**的声音近似 **`da`**；由此建立 **音与撞击事件** 的强关联。
- **由观察推出的 root 义（`da` 层）**：
  - 雨滴从空中到地面的**过程** → **到达**（偏**行为性**）。
  - 雨滴**已落在地面** → **到达**（偏**状态**）。
  - 雨滴与地面**接触** → **堆叠态**（接触/叠合那一相）。
- **`a` 与「另一场景」的合并**：
  - 字母 **`a`** 另有第一音源场景：**婴儿出生时的哭闹声**（近似 `a`）。
  - `da` 里拆出的 **`a`** 与上述哭闹场景的 **`a`**：**符号相同、实质来自不同音源场景**，因**音几乎相似**而采用 **合并模式**：两种场景义**都赋予**字母 **`a`**（具体分析时按语境择用或叠加）。

### `d` 在继承树上的位置

- **`da` 为 root**；**`d` 在 `da` 之下**，**继承**你已讲述的 `da` 所含含义（到达的行为性、到达的状态、堆叠态等；**其余义项待你后续补充**）。
- **演变**：**到达** 衍生 **完成**、**结束**（在继承树上为下游义；具体词内是否取「完成/结束」仍由形参与语境绑定）。

### 3.1) `ta` 与 `da`（同属下雨类音源；强度对比）

- **`ta = t + a`**：**强度偏低的打击声**（相对 **`da`**）。
- **`da`**：**强度偏高的打击声**（水滴撞击地面已定标）。
- **共性**：两者都可承载 **堆叠态** 相关义，也可承载 **结束 / 完成**（例：过去式 **`sent`** 等你已定标方向）。

### 3.2) `s`、`z`、`r`：蛇系音源场景（2026-03）

- **第一场景**：观察**蛇**；**嘶声（sss）** → 字母 **`s`** 一支义位：**观察、感知**（与 **`send`** 等「锁定终点」、**撕裂**等义**并列为多义**，遇词择用或叠加）。
- **`r`**：在你体系中为 **`s` 的浊侧**（**体系内分工**，**非**照抄课堂音系表的机械对应）：蛇从**贴地**到**竖起上身**、**锁定 / 指向**防范对象；带**突然、挺身**；故 **`r` 隐含默认的向上竖起 / 指向**（如**种子顶土出芽**）。与表里 **「`r` 指向目标」**、**`h` 朝向**（**§6.21**）可并读；**`h`/`r` 细分工**仍随词例收（**§9.1**）。
- **`z`**：**与 `s` 浊音对位**；在 **`vase`** 等词中可承载**近 `r`** 的**竖起、冒出、指向外**，与词内 **`as`**「从已定空间内向外冒」并读（**§6.27**）。
- **与 `da`/`ta` 等**：蛇系与雨滴系为**不同音源场景**，在总继承树上**并列**；**父子上挂**待词例增多后再整理（**§9.1**）。

### 术语分流（避免混用）

- **堆叠态**：物理/接触意义上的**叠合、贴合那一相**（文档与口述统一用此词）。
- **量子态**：用于解释**接触⇄脱离**等切换**瞬间**的工具性说法；**不**与「堆叠态」混称。

### 与「词内节点」的关系

- **§4** 里对 **`d`** 的简短标签是**分析用词**；需要追到最底层时，回到本节 **`da` → `d` 继承树**；**`s`/`r`/`z` 蛇系**见 **§3.2**。

## 4) 状态、量、完成：核心节点含义（已定标部分）

- **`a`（量/变量/临界量变位）**：
  - 表示“量”的推进，像变量一样在过程中不断更新。
  - **抽象定调**：**`a` 首先是「量」**；**具象化**时随词/句可读为**空间大小、距离长短、时间长短、程度多少、数量多少**等——**量纲随形参换**，不先钉死某一种。
  - **主重读 `a` 的音与「量」定性**（**2026-03**，与 **`vast`/`vase`/`wall`/`waste`** 对读；**§2.6** 同条）：
    - **`ɑ` / `æ`** → **不定数**；**`/æ/`** 可叠**流体/抽象空间**隐含义。
    - **`eɪ`** → **固定量**（**锁界**，如器皿容积、**钱粮资源一档**，**`waste`**：**§6.29**）。
    - **`ə`** → **单一整体 / 一个**。
    - **`ɔ`** → **围合空间**（**§2.6**、**§6.30**）。
  - 可表示“临界量变到质变的结束位”（达到阈值、接近结束的关键量）。
  - `a` 的读音不同可承载不同义；**`eɪ` 与** **`/ei/`**（**§4.2**）在条件判断上**可并读绑定**；另 **`/ei/`** 常见绑定：
    - 可表达“足够/达到阈值”（在完成态里更稳定）。
    - 也可表达“结构/构造”（在构造/进行语境里可覆盖‘足够’义）。
- **`e`（变化性之量 / 结束焦点 / 整体化 / 产物）**：
  - **`e` 作为「量」**：本性属**变化性**；即使经**打包、赋值**指称某一行为块（如 **`shelt`**），**打包后的 `e` 仍具变化性**。
  - 单靠 `e` 往往不够定态，需**邻接音位/字母**协助（例：**`end`**：`n` 使 `e` **状态化**；`d` 提示**完成性**）。
  - 过程越接近结束，关注焦点越向 `e` 倾斜（结束位权重增加）。
  - 在完成发生时，`e` 可作为**完成的结束产物**：把过程结果打包成一个整体对象（例如订单作为整体）。
  - `e` 可发弱读 `/ə/`，代表“打包/封装为整体”的量。
- **`t`（状态位/状态转移位；接触—脱离；可选出现）**：
  - 用于描述过程中的状态（如没做、做着、做到够量、停止等）。
  - **接触与脱离**；特定情况下可用「量子态」式说明切换瞬间；**堆叠态**与之**分流表述**（见 **§3.1**）。
  - **`t` 非每个词强行插入**：语境需要才成立，亦可承担**打包/赋值**提示。
  - 在 **`vast`** 一类词中，分析块 **`st`**：**视线所能触及的界**（**§2.6**、**§6.26**）。
  - 另：**`ta`/`da`** 强度对比与共性见 **§3.1**。
- **`d`（达到 / 堆叠态 / 完成 / 指向 等）**：
  - **继承树**：见 **§3**（`da` root → `d`；到达衍生完成、结束）。
  - **词内**：在特定词中可表达到、重合、堆叠态、**完成**、**获得/堆叠（非分离）**、**指向/针对**等（由形参与**场景—时态匹配**决定）；与 `e` 的关系以各词已定标为准。
  - **详则**：**§4.1**（`d` 与 `t` 分工、`end`/`ending`、**`send`/`sent`**）。

### 4.1) `d` 的「完成」与「指向」；与 `t` 分工；场景—时态匹配

- **`d` 与 `t` 皆可关联「完成」**，但分工不同：
  - **`d`**：常管**堆叠、获得、到达**一侧；在 **`send`** 中 **`d` 不拿去表「分离/脱手」**，而保留**堆叠/获得发送物**（如 *send a dog* 中**被指定的发送对象**）。
  - **`t`**：可表**分离/脱手**（物到目标点后**交付、脱离发送方**）；故过去式 **`sent`** 用 **`t` 收「到点并分离」**，比形式上的 **`sended`** 更利于**语义分工**（词干 `d` 与词尾「分离完成」不抢位）。
- **`end`（词义核心 = 结束）**：
  - **`d` = 完成** → **`end` 整体：结束更替态**（更替收到收束/完成档）。
  - **例外：进行时**（如动词 *ending*）**仍在更替态**（收束尚在进行），**不按「已结束更替」**读。
  - **`en`**：**不多拆**；进行时与 **更替态** 并行；**`e` 音位**见 **§4.2**（**`ɛ` = `ei`**）。
  - **`d` 的指向**：默认对准「结束」；有宾语 → *end the meeting* **针对**客体。
  - **同形 `ending`**：**名词** = **结束方式 / 结局**；**动词进行** = **正在结束**（详 **§6.10**）。
  - **进行 `ending`**：因 **`en` 更替持续**，**`d` 多取指向**（非「已结束」主导）；与 **`head` / `heading` 中 `d`**：同系，详 **§6.21**、**§6.25**。
- **一般时（概念描述）**：**`d` 多含「完成」义**；若**另有宾语**，进入**场景—时态匹配**：叠加 **`d` 的指向**。
- **`-ed`（如 `ended`）**：标**过去、事件落定**；与词干 **`d`** 默认**不重复造义**（时态壳），若叠加强化另定标。

### 4.2) 音位：`ɛ` 与 `ei`

- 在你体系中：**`ɛ` = `ei`**（**直接等价**，可称转音），用于统一 **`send` / `pend` / `end` 等词中 `e` 的条件判断**（与更替态、`en` 绑定等）。
- **进行体 vs 非进行**（抽象义，**2026-03 定调**）：
  - 当分析绑定到 **`ɛ` 或按 `ei` 处理**的音位条件时：**处于进行体**（如动词 **`-ing`**）→ **`ɛ`/`ei` 表「不断变化」**（更替未封死）；**非进行体** → **「等待状态」**（量停在一档、尚未被进行体搅动；**例外读法碰到再单解**）。
  - **具象化**：可落实为**程度的累加**、**程度是多少**（厚度、重量、标量档位等）。
- **与字母位 `ea` 等**：如 **`head`** 中拼写 **`ea`**（音常 **`/ɛ/`**），进行体 *heading* 下 **`i` 偏生成、`e` 偏变化**，**音位同时作用到量 `e` 与量 `a` 两侧**（详 **§6.21**、**§6.25**）。

### 4.3) 字母重复（暂定：`add`）

- **`add`**：**两个 `d`** 各表**一次获取/叠进**（算术形参 *add 5 and 3*：**前一 `d`、后一 `d` 各取一数入 `a`**，收成结果）。是否升格为通则「**同字母重复 = 同一操作迭代**」，**随更多词例再定**（备忘 **§9.1**）。

---

- **`k`（攻克）与 `c/k`（发起/攻克动作）**：
  - `k` 的基础义是“攻克”，更自然的落点是“攻克某任务/目的要求”（而非生硬说“攻克某动作本身”）。
  - `t` 不一定需要出现在每个单词中，且 `t` 也可能有其他用法（遇到再定标）。
- **`r`（指向）**：
  - 表示目标指向性。动词中常体现为“指向动作目标”；名词化时也可能体现为“概念/对象的指向性”。
- **`o`（方案/规则/含有 等多义）**：
  - 可表示方案/法则/规则（如秩序、流程方案）。
  - 也可抽象为“含有/占据/安置到位”（在不同词中取不同落点）。
  - 也可以表示“完成”（在你后续定标的语境中启用）。

### 元音/辅音与“量”的约束（新增定标）

- **元音才能作为“量”考虑**：`e`、`o` 这类元音可作为“量/整体/单体”的载体；`d` 作为辅音更偏“达到/获取/达成”的动作节点。
- **`o` 的弱读 `/ə/`**：`o` 可发 `/ə/`（与你在 `order` 的 `er`、以及短语 `a dog` 中 `a` 的音同类），表示“一个整体/单体”的打包态（整体化/单体化）。

## 5) 词形/时态：结构承载与意义分工（已定标部分）

- **原形（如 `eat`）**：作为描述模板/形式参数（未进入具体句子语境实例化时，更多是“描述性意义”）。
- **进行时 `-ing`（语法层）**：
  - 表示执行过程，也可状态化修饰名词（英语基本语法）。
  - `-ing` 层统一带来“无限接近目标/进行中未完成”的效果；这不是某个动词本身的问题。
  - `-ing` 后缀的音素拆解（i/n/g）延后分析。
- **过去式 vs 过去分词（句法语境分工）**：
  - **过去式**：强调过去事件完成（状态更确定）。
  - **过去分词**：强调完成后的状态，可形容词化/可挂接在结构上（如 have/be）。
- **时态外置**：如 `did eat`，过去时态转交给 `do` 系统，动词回到原形模板位。
- **场景—时态匹配**：同一字母（如 **`d`**）在**一般时 + 无宾语 / 有宾语 / 进行时**下，可切换**完成**与**指向**等主导义；分析时先绑**句子场景**，再绑**时态体**。

## 6) 理解范例（分析过程快照——用于重置后复现推理）

> 每条范例格式：**形参 → 切分/节点 → 推理链 → 与时态/词性的交互**。目的是**训练推理路径**，不是替代词典释义。

### 6.1 `eat` 族（原形 / -ing / 过去式 / 分词 / 外置时态）

- **形参**：`eat` 作不定式模板；入句后由时态重绑。
- **链（摘要）**：`e` 近停止焦点；`a` 临界量；`t` 状态位；`/ei/` 在 `ate` 表足够量并触发向结束转移；`ate` 过去式下 `t` 为**确定态**（停止吃）；量子叠加仅作**切换瞬间**解释工具。
- **`-ing`**：`a` 不发音则不对 `a` 额外加权；`e`→长元音表持续逼近结束；`-ing` 表进行中。
- **`eaten`**：`en` 状态化完成；`a` 音空则弱化该位。
- **`did eat`**：过去时由 `do` 承载，`eat` 保持模板位。

### 6.2 `make` / `making` / `made`

- **形参**：制作类；边界由词义实例化。
- **链**：`m` 任务边界；`a(ei)` 逼近边界；`k` 攻克要求；`making` 中 `ei` **结构/构造** 覆盖 **足够量**；`made` 中 `d` 表达到/完成；`ei` 两义可并存，抽象/具象双轨表述。

### 6.3 `take` / `taking` / `took` / `taken`

- **主旨**：位置/状态变化（强调「变到哪一档」）；**不**承担「如何理解对象」。
- **链（摘要）**：`t` 抽象为状态转移；`taking` 中 `k` 持续攻克 `ta` 所述任务；`-ing` 无限接近；**完成**与 **`o`**；分词 **`en`** 收束；`took` 中 `ta/to` 表拿到落点，后续 `o` **强化 `k` 侧的攻克完成**；`taken` 中 `ken` 等按你已述收束。
- **注**：`taking` 拼写中无字母 `o`，**不把 `o` 强加进 `taking` 链**。

### 6.4 `order`（下单/订单形参）

- **链**：`o` 方案；`r` 指向；`d` 完成推进；`ord` 过程向完成；`e`（弱读）完成产物、打包整体；名词时 `er` 名格化。

### 6.5 `set`

- **模式**：**锁定 = 开关**；词内**弱过程性**，一般不套量子过程叙述。
- **链**：`s` 锁定；`e` 元素位（**`ɛ`/`ei` 等价**，**§4.2**），可为 0/1/多（含空集）；`t` **仅二值**：开/关或真/假；时态形式上常 **`set`** 不变（你已定标）。
- **`st`（理论块）与 `set` 对读（义理，非词源）**：**`s` 锁槽 + `t` 落定** → 与上列 **`set`** **同脉**。词尾 **`st`**（如 **`list`**）可承「**逐项 set 到预定格**」。**`si` / `is` 顺序对位**见 **§6.42**。

### 6.6 `side` / `cide` / `decide`

#### `side`（保留原快照）

- **`s`**：锁定目标点（如视线未到点先钉住）。
- **`i`**（音 **`/ai/`**）：向锁定点**收束成单一整体**。
- **`d`**：**重合**。名词：`d` 重合即完成，`e` 产出元素；动词：`d` 仅重合，**完成由时态**（如 *sided*）。

#### `cide`（干；与 `side` 并行，拼写选 *decide* 降歧）

- **`c`（读作 /s/）**：动作形态为**锁定一点的移动**——如**笔尖贴在纸上**，**焦点在笔尖**，拖动时**尖始终是焦点**；类推**切肉**（刃口）、**视线扫过**。抽象：**刻痕、印记、切割类**行为。
- **`i`**：**漫延、拖动**——**连续、不间断**的一串运动；音 **`/ai/`** 时再叠**导向**（如 **guide**：刀路/视线**按心里已定方向**走）。
- **`d` + `e`（词干 `cide` 内）**：**`d`**：**堆叠态/贴合/落定**——**切分落档**（整体与切下的那一块**各归其位**）；**`e`**：随词可接**结果/名物**侧（与 **`cide` 衍生裁决**并读）。
- **衍生义**：**裁决**、**杀**——极端形参下「**一刀划开，两边定性**」。

**「决定」义深化（非「裁两半再选」）**

- **不是**：先切成两半，再**犹豫选哪边**（那是未决）。
- **而是**：像**从整盘里分出一份**——**整体仍在**，**切出**的那一块**份量/切法事先已知**；**`i` 的导向**表示**该如何切、切多少**在动刀前**已定**，下刀是**执行已定案**。
- **白话**：**决定**＝**份额与刀路先在心内定好，切割只是把该取的那一份从整体卸下来**。

#### `decide`（`de` + `cide`）

**例句**

- *She **decided** to leave.*

**切分**：**`de` + `cide`**。

- **`de`**：**获取**待判断、待处理的那件事（从多可能/混沌里**抽出**要落锤的对象，与 **`depend`** 系 **`de` 同系不同形参**，**§6.12**）。
- **`cide`**：按上节 **导向切割** 裁出**结果**——**执行**已定方案，**落定**为决定。

**白话**：**`de` 把「要判的那盘」端清楚，`cide` 按心里已定的那份下刀——决定落成。**

> **备忘（§9.1）**：早年对话里**多米诺骨牌链式**具象可作**可选锚**，与上列**分餐式决定**并行；未单独立节。

### 6.7 `act` / `acting` / `acted`

- **链**：`c` **攻克** `a`（`a` 可为任意事/事件/抽象量）；`t` 为动作状态位；**`-ing` / `-ed` 绑定 `t`**：`acting` 正在动作；`acted` 停止工作/完成段攻克。

### 6.8 `shell` / `shelt` / `shelter`（自然义为先；技术义为后衍）

- **义项顺序**：**贝壳等自然外壳**为基底；炮弹壳、计算机 shell 等为**后衍形参**（借「外层硬罩」骨架），不在此挤占自然义优先权。
- **`shell`（形参：贝壳）**：
  - **`sh` /ʃ/**（**先整体**，后再分 `s/h`）：**遮盖**，内外分隔。
  - **`e` /e/**：**向闭合位之变**（两侧朝合拢、收口）。
  - **`ll`**：**仅闭合口缘轮廓**（两瓣贴合的那一圈界线），**不含**壳面花纹纹理；**与 `wall`/`well` 等处 `ll`（沿边连续铺开 / 自源溢出 / 成双）形参不同**，见 **§2** 表 **`ll`**、**§6.30**、**§6.31**。
- **`shelt` 与 `shell`**：两缘 **`ll` 重合**后并作**单 `l`**；**`t` 提示重合**；行为呈**开关性**（遮住 / 未遮住），与 **`set` 类锁定开关**气质相通。
- **`shelter`**：**`e` = `shelt`**（`shelt` 为**行为性**描述，故 **`work` 式亦可动可名**）；**`-er`** 上接时：**动词**（庇护行为）易理解——同类 **`work`/`worker`：行为性词根 + `er` 名格/施事**。

### 6.9 `send` / `sending` / `sent`

- **`s`**：锁定**发送的目标终点位**（目的地/结束位那一档）。
- **`en`**：**由时态决定**（见 **§2 表 `en` 行**）：
  - **进行时**（*sending*）：**`en` = 更替态**，**不断朝目的地变化**（未稳态完成）。
  - **过去**：**`en` = 稳定态**，**完成发送**。
  - **一般时**：**概念描述态**，叙述仍带**完成向**。
- **`d`**：在 **`send`** 中落实**完成**一侧，且可表**获得**——**发送物**（如 *send a dog*：**被指定/获得的那件发送对象**）；**`d` 在此不表分离/脱手**。
- **`sent` vs `sended`**：形式上 `sended` 可想象，但 **`d` 与 `t` 皆可表完成**；此处 **`d` 保留堆叠/获得**，**分离/到点脱手**交给 **`t`** → **`sent`** 更利分工（见 **§4.1**）。

### 6.10 `end` / `ending` / `ended`

- **词义核心**：**结束**；**`d` 的指向默认对准「结束」**；有宾语则对准**那件事的结束**。
- **`end` 与更替态**：**`d` = 完成** → **`end` 结束更替态**；**例外**：动词进行 *ending* **仍在更替态**（见 **§4.1**）。
- **`en`**：**不多拆**；**进行时**（*ending*）：**更替态** → **`e` 持续变化**；**`ɛ` = `ei`**（**§4.2**）。
- **`d`**：
  - **一般时（概念）**：多含**完成**义。
  - **一般时 + 宾语**（*end the meeting*）：**场景—时态匹配** → **`d` 的针对/指向**（对准要结束的客体）。
  - **同形 `ending`**：**名词** = **结束方式 / 结局**；**动词进行** = **正在结束**。
  - **进行 `ending`**：因 **`en` 始终更替**，**`d` 取指向义**（非「已结束」主导），与 **`head` 中 `d`**：**同系关联**，`head` **专讲待定**。
- **`ended`**：**`-ed`** 标过去、事件落定；与词干 **`d`** 默认**不重复展开**（时态壳），若叠加强化另定标。

### 6.11 `pend` / `pending` 与 `pe` / `pen`（`p` 的产出落在 `e` 上）

- **`p`**：**分开/分离**（常带爆发性）；衍生 **部分**；抽象 **变化**（一分则有变）。
- **`pe`**：**`p` 过程的产出**落在 **`e`（结束性变化）**上 → 即「**分离的变化**」；用语见 **§2.2**（**不说「修饰」**）。
- **`pen`（笔；与 `pend` 中 `pen` 核同逻辑）**：**墨/颜料不断分离**到书写材料；**`n`**：**印迹、笔迹、留痕**（与 **`line`/`ink` 中 `n`** 同系）；在**名词（笔）**与**进行时**语境下可处 **更替态**（与 **`pend` 核**对齐时）。
- **`pend`**：**结束分离**（分离进入收束相）；**更替态结束**，**不断分离停止**（用 **pending 逻辑**统读，不用「变化停住」旧说法）。
- **`pending`（`pend` + `ing`）**：**正在结束分离**——**悬挂、尚未彻底分离**；具象可联想**黏稠物下落、变干而越来越慢的分开**（分离在进行且减速）。
- **与 `end` 分工**：**`end`** 里 **`d`** 完成 → **结束更替态**（进行时 *ending* 为例外，仍更替），见 **§6.10**、**§4.1**。

### 6.12 `pend` 前缀族：`append` / `depend` / `impend`

- **`pend` 核**（复述）：**结束分离**；通俗 **悬挂至尾**（接缝/末端收口）；**`pending`** 见 **§6.11**。

#### `append A to B`（附加到尾）

- **`a` 读 `/ə/`**：暗示 **格内已有单体**；**`ap`**：**分离或分配 `A` 进入 `a`**（**输入向**；与 **`pa` 输出向**对举），**`a` 中含 `A`**。
- **`pend`**：**悬挂至尾**——接到**当前链/对象的尾巴**；**`B`** 为**承挂背景**，由 *to B* 给定，**分析链里可不另开 `B` 量位**。
- **末位 `d`**：**完成/到位**（附加落定）。

#### `depend`（依赖）

- **切分**：**`de` + `pend`**。
- **`de`**：从堆叠/多可能中**抽出、落定挂靠点**（与 **`decide` 系 `de`** 同系不同形参）。
- **`pend`**：**悬挂**——**挂在外置支点**上（非 *append* 的「链尾加块」）；**`on B`** 给定支点时可**不显式再开 `B` 量位**。

#### `impend`（迫近、悬而未落）

- **切分**：**`im` + `pend`**。
- **`i`（音 `/i/`）**：**漫延**。
- **`m`**：**边界**。
- **`im`**：**漫延至边界**。
- **`im` + `pend`**：**蔓延到边并悬停在那**——**迫在眉睫、尚未落下**。

### 6.13 `iterate`（遍历 / 迭代；编程类比快照）

> 拆分以**讲清过程**为先，字母切法可按教学需要微调。

- **形参**：**`iterate A`**——对 **A** 做可重复遍历，每轮可对应一次**质变**；**同一个 `e` 槽**经多轮 **`a` 累积**被改写。
- **流程（你已述）**：
  1. **`i`（音 `/i/`）**：**拽取**——把 **A** 拉进 **`i` 量**（可抓着走）。
  2. **`t`**：把 **`i` 量**转成**结束性变化位 `e`**；**`e` 弱读 `/ə/`** → **一轮质变单元**。
  3. **`r`**：**指向/驱动**开始量 **`a`**——让 **`a` 做够变化**以完成本轮 **`e`**；**`a` 音 `/ei/`**（**`ɛ`=`ei`**，**§4.2**）→ **足够完成本轮质变的量变**。
  4. **`a`**：**循环累积**——若质变 **n** 次，则 **`a` 走 n 轮**堆够量变。
  5. **再经 `t`**：**输出**质变后的 **`e`**（**身份同一 `e`**，内容被 **`a` 改写**）。
- **编程类比**：数组 **A** → **`i` 抓成可遍历的 `e`** → **`r` 指挥 `a` 循环 n 次** → **终态仍是那个 `e` 位、已是 n 次质变后**。

### 6.14 `it` / `bit`

- **`it`**：**`i` + `t`**
  - **`i`（拽取）**：从可分背景里**拽出其中一个**。
  - **`t`**：**状态化**且已在**结束位**——**无需再转成别的量**；故 **`it`** 易**名格化**为代词「它」。是否再作形/副用法**看词义是否合适**。
- **`bit`**：**`b` + `it`**
  - **`b`**：**分离、分开**。
  - **`it`**：同上 → **揪出其中一小块并钉成单位**（一小点、一口、一片等）。

### 6.15 抽象块 `mit`（非独立词）

> **`m` + `it`**；出现在多词词干末尾（**`-mit`**）。

- **`m`**：**边界**——在此模式中特指 **满足边界要求 / 符合边界要求**（条件达标、与边对齐）。
- **`it`**：同 **§6.14**——**拽取其中一个** + **`t` 状态化** → 收成**可指称的那一个单元**。
- **过程（具象锚）**：如**屋檐滴水**——在边界处**攒够分离条件** → **产出一滴**作为 **`it`**。
- **抽象句**：**满足（或符合）边界要求后 → 获取并形成 `it`。**

### 6.16 `admit`（`ad` + `mit`）

- **切分**：**`ad` + `mit`**。
- **`ad`**：
  - **`a` 弱读 `/ə/`**：**一次足够的质变性变化**。
  - **`d`**：**获取**——**把 A 获取入 `a` 这一量位**。
  - **合成**：该质变使 **边界要求可被满足**，为 **`mit`** 铺路。
- **`mit`**：见 **§6.15**——边界要求满足后，收成 **`it`**（承认的内容、准入结果等）。
- **譬喻**：
  - **承认错误**：再三思考 → **质变够格** → **`ad` 把「错误事实」纳入 `a` 量** → **`mit` 落成「承认」之 `it`**。
  - **凭票入场**：**门票** = **质变/量变条件** → **`ad` 侧完成纳入** → **边界满足** → **`mit` 产出「准入」之 `it`**。

### 6.17 `perfect`（`per` + `fect`）

- **`p` 与 `b`/`f`**：同属**分开**；**`p` 另强调部分** → **`per`**：**就每一部分**。
- **`pe`**：**部分上的结束性变化**；**`e` `/ə/`**：**已质变**；**`r`**：此处用 **「导致」** 很贴——**导致**后续名格/坍缩（蛋糕切块**量子态→落定**可作具象锚）。
- **`fect`**（见 **§2.4**）：**`fe`** 管**质变**；**`ct` 坍缩** → **最终质变**；**`fact`** 重**量变→质变**，**`fect`** 重**质变点**。
- **两个 `e` 不同**：**首 `e`** 锚**一次质变**；**后 `e`**（音可 **`/ei/`**）→ **多次质变**，意象 **蝴蝶效应**。
- **`per` + `fect`**：**任一部分上的质变都会 `导致` 作用对象经历 `fect` 侧质变**，次数 **一个或多个（概数）**。
- **「完美」**：**正向褒贬** = **约定俗成**，非音素必然。

### 6.18 `affect`（`af` + `fect`）

- **`af`**：**`f` 提示 `a` 已达质变量**——**一次导致质变的量**；**`a` 本重过程**，**`af` 把过程收死在够触发阈值**；**`a` 音**常近 **`/ə/`**（分析时**写 IPA**）。
- **`A affect B`**：**获取 A 入 `a`**，**`B`** 为**作用对象**；再接 **`fect`**（**质变点 + `ct` 坍缩**）。
- **`ff`**：**双写强化 `f`** 或**两波分离**，**待定**。
- 与 **`effect`**：**§6.19**。

### 6.19 `effect`（与 `affect` 对读）

- **主轴**：**`e` 重结果**（对 **`affect` 的 `a` 重过程**）。
- **试读**：**`e`（结果位）+ `f`（分离出、抽出成独立果）+ `ect`/`fect` 系**——**质变已坍缩后的那一果/那一态**，可被**指称、记账**（名物化语感）。
- **`affect` / `effect`**：同一 **`fect` 系**上，**过程端 vs 结果端**的分工。

### 6.20 `permit`（`per` + `mit`）— 先例句，再拆字

**切分**：**`per` + `mit`**；**`-mit`** 见 **§6.15**。与 **`admit`** 一样，末尾都是「**边界对上了，落下一件具体的事**」；差别在开头：**`ad` 像「从外面收进来」**，**`per` 像「一条一条对规矩，对完了才放行」**。

---

**例句 1（骨架：A permits B to …）**

- *The school **permits** students **to use** the hall after class.*

**白话**：学校是**说了算的一方**；它对准许的事（课后用大厅）**按校规一条条对齐**（该满足的满足），对齐之后，这件事就从「不许」变成「可以」——**落下一个明明白白的「准许」**。

**结合 `per` + `mit` 看**：

- **`per`**：**掌权方心里那套规矩**往往不止一条；**`p` 带「分部分」**，就像**逐项打勾**；**`r` 导致**这些勾最后要**接到**后面的 **`mit`**。
- **`mit`**：**条件对齐了**，就**收成一件可指称的事**：这里收成的是「**学生被允许去用大厅**」这一档（**§6.15** 里「屋檐滴水落成一滴」那种**落定成一个单位**的感觉）。

---

**例句 2（被动，看「边界不通过」）**

- *Smoking is not **permitted** here.*

**白话**：这里的规矩是「**烟这件事别过关**」。**`mit` 那一关过不去**，就没有「准许抽烟」那一**件**事落下来。

---

**例句 3（名词：准许证本身）**

- *You need a work **permit**.*

**白话**：**纸/证**就是已经把「边界对齐」**打包好**的那**一件**东西——带在身上，别人一看就知道你**过了哪几道关**。**同一个词根**，动词是「**让过关**」，名词是「**过关后给你的那一张**」。

---

**和 `admit` 放一起看（仍用白话）**

- **`admit`**：常像「**把话/人/事实收进来**」——**收进来**之后，**认**了、**让进**了（见 **§6.16**）。
- **`permit`**：常像「**规矩逐项对完，盖章放行**」——**不是强调从外面捞进来**，而是强调**每一部分都对上**，再**落到「可以」这一件**。

### 6.21 `head`（动词：朝某处去 / 开往）

**例句**

- *We **headed** home after the show.*
- *She is **heading** for the coast.*（进行体另见 **§6.25**）

**切分**：**`he` + `ad`**（字母 **h-e | a-d**）。

**白话 + 字母**

- **`he`**：**`h`** = **指向、朝向**；**`e`** = **目的地那一档**（要去的那头在链上的结果侧锚点）。
- **`ad`**：**往前拱、距离往 `a` 上累**；**`d`** = **堆叠**（一步一叠）。
- **`a` / `e` 分工**：**`a`** = **量变**（路程、步数）；**`e`** = **质变点**（到没到「算在朝那儿去了」那一跳）。
- **`ea`**（拼写 **e+a**，音常 **`/ɛ/`**）：与 **§4.2** **`ɛ`=`ei`** 并读；进行体里 **`ei`** 表**持续在生**，**`i` 生成、`e` 变化**，**音位同时托住量 `e` 与量 `a` 两块**（两块都在动）。

---

### 6.22 `add`（算术：把两数并到总量）

**例句**

- ***Add** 5 and 3.*

**切分**：**`a` + `d` + `d`**。

**白话**：**两个 `d`** 各**获取**句子里**前后两个数**，**都放进 `a`**（量变槽）；说话人若盯**结果**，收成就在 **`a` 上落到 8**。与 **§4.3**、词内 **`ad` 推进 + 堆叠** 气质可并读：**多一颗 `d`** → **多一次「把旁边的数也叠进 `a`」**，对准**二元运算**形参。

---

### 6.23 `advance`（推进；可含阶段、终点）

**例句**

- *The army **advanced** toward Paris.*
- *The project **advanced** to the next phase.*

**切分**：**`ad` + `van` + `ce`**（**a-d | v-a-n | c-e**）。

**白话**

- **`ad`**：**推进**（与 **`add`/`head`** 里 **`ad` 往前拱**同系；**词内 `ad`** **不必**处处等同前缀块 **`ad`（`admit`）**，见 **§2** 表注）。
- **`van`（本词内分析块，非英语词 *van*「货车」义项）**：**进程中的阶段性变化**——**每进驻一城 / 每完成一大阶段**算**一个 `van` 档**；**`v`** = **`f` 浊音**，**分开、分离**，气质**波状、有节奏**（细则随词补）。
- **`van` 里的 `a`**：只扛**本阶段的量**；整条路上的**总量**由**全词 `a` 的累计**托住（与 **§4** **`a` 具象维**一致）。
- **`ce`**：**锁定目标的结束点**（巴黎、交付里程碑等）；与 **`c`/`ce`** 系（**§2.4**、**§2** 表）并读。

**备忘**：**`advance` 内 `van`** 与 **单词 `van`**（**§6.24**）是**同一字母串 `v-a-n`、两套形参**，见 **§9.1**。

---

### 6.24 `van`（单词：厢式车 / 有壳空间）

**例句**

- *A delivery **van** pulled up.*

**切分**：**`v` + `a` + `n`**。

**白话**

- **`v`**：从**分开、分离**起，推到**空间**——像**波起波伏**，**凹槽、槽形空档**。
- **`va`**：**`a`** = **这块空间有多大**（尺度、容积感）。
- **`n`**：**内部**——**外壳围出来的一膛**（车厢、厢体）。
- **仅 `va`、不收进 `n` 那一壳的读法**（分析上对比用）：可想象**非常大乃至无边际**的空间（无壳收口）。

---

### 6.25 `heading`（`head` + 进行体）

**例句**

- *They are **heading** out.*

**白话**：骨架同 **`head`（§6.21）**；**`-ing` 进行体**接上 **§4.2**：**`ɛ`/`ei` 条件**下表**不断变化**，**更替未封死**——**朝目的地去**这件事**还在路上**，**量 `a` 与量 `e` 两侧**随进行体**持续在动**。

### 6.26 `vast`（巨大；不定数 + 视程）

**例句**

- *a **vast** plain* / *a **vast** ocean of data*

**音**：美音常 **`/æ/`**，英音常 **`/ɑː/`**；在你体系中**二者皆属不定数**（**§2.6**），与 **`vase` 美音 `eɪ` 锁界**对读。

**切分**：**`v` + `as` + `t`**（字母 **v-a-s-t**；**`s`** 在 **`as`** 与 **`st`** 上**交叠承义**）。

**白话**

- **`v`**：**分开→空间/波槽**（**§2** 表）。
- **`as`**：**`a` 的量扩到可感知的边** + **`s` 观察、感知**（**§2.6**）；**`/æ/`** 时可叠**流体/抽象场**隐含义。
- **`st`**：**视线所及之处**。
- **合起来**：**大到感知与视线能罩到的那片边界**。

---

### 6.27 `vase`（花瓶；锁界空间 + 向外冒）

**例句**

- *She put flowers in the **vase**.*

**音与义理（你体系内判读）**：**英音**常 **`/ɑː/`**（类不定数） vs **美音**常 **`/eɪ/`**（**固定量、锁界**）。**器皿**形参宜 **「腔体容积已定」** → 你认 **美音 `eɪ` 更贴义理**；**英音 `ɑː`** 则与「有界容器」**易拧**，标为**体系内待对读之异读**（非词典对错）。

**切分**：**`v` + `a` + `s` + `e`**。分析块：**`va`（v+a）** 与 **`as`（a+s）** 在字母 **`a` 上交叠**；字母 **`s`** 读 **`/z/`** 时走 **`z`** 义位（**§2** 表、**§3.2**）。**末位 `e`**：**弱收尾/名物化外壳**，不抢 **`as` 主链**。

**白话**

- **`va`**：**已定的一膛空间**（瓶腹容积）；**`eɪ`** 把 **`a` 锁成定量**（**§2.6**）。
- **`as`**：从**这膛已定空间里向外冒**——**花梗、花头伸出瓶口**，**有一部分在界外**。
- **`s`→`/z/`→`z` 义位**：**近 `r`** 的**竖起、冒出、指向外**（蛇系浊侧，**§3.2**），托住「**伸出瓶口**」那一截。

---

### 6.28 `rise`（升起；试读，接蛇系 `r`）

**例句**

- *The dough **rose** overnight.* / *He **rose** to his feet.*

**白话**：**竖起、从低位到挺身**——贴 **§3.2** **`r`**：**蛇挺身、种子顶土、向上指向**；与 **`h` 水平朝向**（**§6.21**）可对照。

**切分**：**`r` + `ise`（干待定）**——**`ise` 字母级终稿随词例再收**（**§9.1**）。

### 6.29 `waste`（浪费；`wa` + 外露 + `te`）

**例句**

- *Don't **waste** water.* / *They are **wasting** time.* / *The effort was **wasted**.*

**切分**：**`w` + `a` + `s` + `t` + `e`**。分析块：**`wa`**（**`w` 合围 + 所含 `a` 量**）+ **`as`**（字母 **`a` 与 `s` 交叠**时按 **`as`** 读）+ **`te`**。

**白话**

- **`w`**：**吾合、包裹、包围**——先有一层「**拢在手里的总量**」（钱、水、时间等资源，**§2** 表）。
- **`wa`**：**抽象**＝**被围入、被掌握的那一份量**；**`waste`** 里 **`a` 音 `/eɪ/`** → **定量**（**§2.6**：锁在一档可数的资源池）。
- **`as`**：**量往外露**——露出去就**不再属于你这份库存** → **耗掉、浪费**（与 *vast* 里「扩到视程」**不同形参**）。
- **`te`**：**`e`**＝**被浪费掉的那一份量 / 浪费这一果**。**进行时 *wasting***：**一直漏、一直烧** → **结果尚未钉死**（与 **§4.2** 进行体气质并读）。***wasted***：**经 `t` 落定，把 `e` 输出**——**浪费已成事实、量已收成在 `e`**。

---

### 6.30 `wall`（墙；围膛 + 沿边 `ll`）

**例句**

- *a garden **wall***

**切分**：**`w` + `a` + `ll`**（**`all`**＝**`a` + `ll`**）。

**白话**

- **`w` + `a`**：**`wa`**——**围出一膛空间**；**`a` 音 `ɔ`** → **围合空间**（围栏、内膛；**不必**绝对闭，**§2.6**）。
- **`ll`**：**沿边界连续铺开、成序列**——墙段**绕着**内部那膛 **`a` 空间**排过去（与 **`shell`** 口缘 **`ll`** **不同形参**，**§2** 表）。

---

### 6.31 `well`（井；源头 `e` + 溢出 `ll`）

**例句**

- *draw water from a **well***

**切分**：**`w` + `e` + `ll`**（**`wa` 气质**在 **`w`**；源头在 **`e`**）。

**白话**

- **`w`**：**合围**——**井筒**把**地下水膛**圈住；**源头在地下**（你定的「源头被围」具象）。
- **`e`**：**井的源头、涌出的那一头**（**结果/变化侧**与 **`w` 围膛**对读）。
- **`ll`**：**自源头连续溢出、延绵扩散**——水**不断**从 **`e` 源**经 **`ll`** 漫出来（**§2** 表 **`ll`** 义项③）。

---

### 6.32 `bull`（公牛；试读：`ll`＝成双）

**例句**

- *a **bull** on the farm*

**白话**：**`b`**（分开系，**§2** 表）+ **`ull` 中 `ll`** → **成双、长条对称**——**两只角**竖在头**两侧**（**§2** 表 **`ll`** 义项②）。**`bu-` 干**细拆**随词例再收**（**§9.1**）。

### 6.33 `wealth`（财富；与 `worth` 同脉）

**例句**

- *a nation’s **wealth*** / *spiritual **wealth***

**逻辑**：与 **`worth`**（**§6.34**）**同脉**——**源头 → 攒够 → 列出 → 用 /θ/ 收进内膛存住**；**`wealth` 多一维**：带**阶段性、过程性**，强调**财富的积累**（一截一截堆出来的家底），不是对**单回收获**的估量。

**切分**：**`we` + `a` + `l` + `th`**（字母 **w-e-a-l-t-h**）。

**白话 + 字母**

- **`we`（块）**：**`w`** 在此兼 **务**——**从事工作、事务**（如 **`work`**），不是空泛的「围」；**`e`** 取**收成/结果侧**。**合成**：**通过干活、办事汇出来的那一脉「财富源头」**——工资、家业、**精神积累**都可挂这一膛。
- **`a` 与 `e` 联动**：你定 **「`a` 变，`e` 也跟着变」**；在此词里**并读 **`eɪ`/`ei` 条件**（与 **§2.6**、**§4.2** 可绑）→ **积攒到足够的一档量**（够格叫「富」的那一坨）。
- **`l`（单写）**：**列出（list）**——**一条条摆开、入账**（与 **`ll` 双写**的墙/井/壳**形参不同**，**§2** 表）。
- **`th`（音 /θ/）**：**合在内部、存储**（钱粮入库、观念进脑）；与 **`worth`** 同音位骨架（**§2** 表 **`th`**）。

**与 `worth` 对读**：**`wealth`**＝**过程 + 积累**；**`worth`**＝**单次收获的评估**（见下）。

---

### 6.34 `worth`（价值；单次收获评估 + 思维判断）

**例句**

- *The ring is **worth** a lot.*

**切分**：**`w` + `o` + `r` + `th`**（**`th` 音 /θ/**，**§2** 表）。

**白话**

- **`w`**：相对 **`wall`/`wa` 的合围**，此处偏 **获取 / 获得**——把成果**拢到名下**。
- **`o`**：**结果**；由 **方案做到头、收成**衍生（**`o` 方案/秩序** 系里的**结案成果**）。
- **`r`**：**指向**——把链**导到** **`th`**。
- **`th`（/θ/）**：
  - **收纳义**：**合在内部**——卵中雏鸟、货入仓、**思想在脑中**；**存住了**才谈得上**值**、**值得记**。
  - **思考义**（补）：与 **`think`** 系气质相接——**值不值、划不划算**首先是**脑子里过一遍的判断**，不是只讲物理仓储。

**与 `wealth` 对读**：**`worth`**＝**对一回收成/单档的估价**（**评估**）；**`wealth`**＝**带阶段的过程 + 强调积累**（**§6.33**）。

---

### 6.35 `terminate`（终止；`ter` + 渐进 `te`）

**例句**

- *We decided to **terminate** the project.*

**切分（字母位骨架）**：**`t` + `e` + `r` + `m` + `i` + `n` + `a` + `t` + `e`**（**`-ate`** 动词化尾；**中间 `min`/`mina`/`inate` 等**与 **`mit`/`im`/`na`** 的**精细分工**可随更多词例再收）。

**白话（体系内）**

- **`ter`（块）**：**获取**那桩**待终止**的事务（**§2** 表）；与 **`m` 边界**并读：**边界条件满足后，事收成到 `i` 档**，再进入**终止程序**。
- **首段 `te`**：**`t` 状态/转移** + **`e` 结果侧** → **过程往终局耗散**；**未用 `-ed` 封顶**时偏**渐进、进行中的收尾**（与 **`-ed` 一次落定**对读，**§2** 表 **`t`**、**§4**）。
- **`na`/`ne` 对读**（**§2.7**）：**多段各自收到 0**（停工、解约、撤具…）**合成项目终 0**；**不必**读成「从存在里抹除」。

**与 `determine` 对读**（**§6.36**）：**terminate** 重**关掉、结束**；**determine** 重**裁定、定案**。

---

### 6.36 `determine`（裁定；`ter` + `ine` 短路径）

**例句**

- *The court will **determine** whether he is guilty.*

**切分（骨架）**：**`d` + `e` + `t` + `e` + `r` + `m` + `i` + `n` + `e`**（**`-ate` 类尾**略同 **§6.35**）。

**白话（体系内）**

- **`de`（块）**：**堆叠中抽出/落定**（**§2** 表；**§6.6**、**§6.12**）。
- **`ter`（块）**：**获取**待**裁定**的对象（与 **§6.35** 同块，**义随形参**换为「定案」而非「关停」）。
- **`ine`（块）**（**§2** 表、**§2.7**）：**`i` 上尚有非零未定** → 经 **`ne`** → **0**；**末位 `e`** 承**裁定后的结果档**（**结论收到零档/定案**）。**走 `ne` 短路径**，**不**依赖 **`na` 长链收敛**的叙述。

**与 `terminate` 对读**：见 **§6.35**。

---

### 6.37 `not`（完全无；`n` + `o` + `t`）

**例句**

- *That is **not** true.*

**切分**：**`n` + `o` + `t`**。

**白话（体系内）**

- **`n`**：**收敛**（**§2** 表、**§2.7**）——**量档往「无」收**。
- **`o`**：**形似 `0`** + **完全、到头**（**§2** 表）→ **「完全的无」**；**不是**只剩半档的模糊否定。
- **`t`**：**落定、封死**该否定（**§2** 表 **`t`**；与 **`d`/`t` 完成壳**并读，**§4.1**）。

**何以取 `o` 而非别元音**：**义理上 `n`/`t` 已能托「收到无」**，***not*** 再叠 **`o`**：**字形锚 `0`** + **语义「完全无」**，与 **`na`→0、`ne`＝0** 的布尔读法**同脉**（**§2.7**）。

### 6.38 **`*nada`（西班牙语）与 `nad` 理论块**

> **义理锚 / 跨语参照**，**不是**英语词源证明。

- **西班牙语 *nada***（什么都没有）：与 **`nad` 块**（**§2** 表）**对读**——**`na` 向 0** + **`d`/`t` 完成壳**把 **0 钉死**；口语里「**nada**」即 **「收到空档且落定」** 的气质。
- **与 `not` 对读**（**§6.37**）：***not*** 走 **英系 `n-o-t` 三音分工**；***nada*** 走 **罗曼语系另一套拼合**，**可在「零/无」这一档上互照**，**不必**强行同一词根。

### 6.39 `line`（线；队形 / 笔画）

**例句**

- *Please **line** up.* / *Draw a straight **line**.*

**切分**：**`l` + `i` + `n` + `e`**。

**白话（体系内）**

- **`l`**：**连续、持续**地形成**细长轨迹**的那种运动（沿窄带延伸）。
- **`i`（音 `/ai/`）**：**`i`** 表**一次变化**；**`ai`** 表**循环式、可重复**的变化；**字母 `i`** 另可托**单一体 / 单一整体**。
- **`in`**：**`i` 在变**时，**`n`** 表**留下印迹**（轮廓、阴影、可辨认的「点」）。
- **`ne`（末段 `n`+`e`）**：**`n`** 印迹之后 **`e` 结束位** → **队排到尾 / 笔画到头**，与 **`ne`＝零档收束** 布尔读法**可对读**（**§2.7**）；**非**声称凡 **`ine` 块**皆同此词（**`ina`/`ine` 未定稿**见 **§9.2**）。

**形参 A：排队** — 每人 ≈ **`n` 的一个印迹点**；**`i` 的每一次变化** ≈ **定下一人位置**；**队尽** ≈ **`ne`**。

**形参 B：划线** — **`i` 的每一次变化** ≈ **沿表面再移一小段**；**`n`** ≈ **移动中留在面上的迹**（线即迹的累积）。

---

### 6.40 `list`（清单；连续列出 + `st` 落定）

**例句**

- *Make a **list** of names.* / *The apples were **listed** to the ten plates.*

**切分**：**`l` + `i` + `s` + `t`** → 骨架：**`li` + `st`**（**`st` 块**见 **§2** 表、**§6.5**、**§6.42**）。

**白话（体系内）**

- **`li`（块）**：**连续输出 `i` 的变化**；统一抽象下**「变化」**＝任意**行为或状态**变化。**`l`** 强调**连续地输出**后面那一串 **`i`**。
- **`i` 音 `/ɪ/`（本词）**：走 **`/i/` 系**——**每次输出一单体变化** → 连续过程里**一节一节、带节律**（例：**水滴**：**每一滴**＝一次 **`i`**）。与 **`line`** 中 **`/ai/`** 对读：**`/ai/`** 可把**首滴至末滴**合成**一个整体 `i`**；**`/i/`** 则**一滴一 `i`**。
- **双元音 `ai`（与 `list` 对读时）**：**挨着、相靠**的输出——**可有间隙**，也可**堆叠**。
- **`st`**：**`s` 锁定位 + `t` 落定** → 义理 **≈ `set`**（**§6.5**）——**每一项列到预定格 / 槽位**。**例**：盘位已设，苹果被 **`list`** 到各自位置 → **`is` 向锁定位** 叙事可与 **`li`+`st`** 并读（**§6.42**）。

---

### 6.41 `sit`（坐定；`si` 块）

**例句**

- *She **sat** on the chair.* / *Please **sit** down.*

**切分**：**`s` + `i` + `t`** → 首段 **`si`** + **`t`**。

**白话（体系内）**

- **`si`（块）**：锁定**当前**这一次 **`i` 变化**——**身体落到座位上**的那一档被**封死**（姿势/位置**就地锁住**）。
- **`t`**：**状态转移 / 落定**（与 **§2** 表 **`t`** 并读）。

---

### 6.42 **`si` / `is` 顺序对位；`st` 与 `set`（备忘）**

> **推理用骨架**，**不是**英语构词法上的普遍历史词源。

- **`si`**：**先锁「当前这一次 `i`」** — 变化**停在此点**（**`sit`**，**§6.41**）。
- **`is`**：**顺序对翻** — **槽位/锁定位已先设**，**`i` 的变化是朝那位去**（**对齐到预定格**）。与 **`list` 入位** 并读（**§6.40**）。
- **`st`**：**`s` + `t`**＝**锁槽 + 落定** → 与 **`set`** **同脉**（**§6.5**）；**`list`** 尾 **`st`** 承「**列项 set 到位**」。

## 8) 标准分析流程（每次分析建议输出的结构）

1. **例句**：先写 **1～2 句典型英文**，标明本次义项（可与 **§1「例句优先」**一致）。
2. **白话剖析**：顺着例句说明**谁对谁、发生了什么**；必要时再点出 **`A permits B to …`** 这类骨架。
3. **切分与字母位**：按字母位为主，必要时写 IPA；若见 **`a`+辅音 / 辅音+`a`**，先走 **§2.5**。
4. **（可选）节点与因果链**：需要收束时再写「量 / 完成 / 边界」等——**详细模板句可放到将来「抽象专章」**，此处能省则省。
5. **覆盖说明**：若存在多义（如 `ei`），写明本次主导义。
6. **多义并用**：若一次启用多个义位，逐条说清在句中的分工（见 **§1**）。
7. **卡点提问**：链条确实卡死时再提一个关键疑问。

> **旧版 8 步**里「抽象版 + 具象版」等条目并未废除；**默认顺序改成「例句 → 白话 → 再抽象」**，与 **§1** 一致。

## 9) 待补 / 搁置（避免重置后误当已定标）

### 9.1) 2026-03 对话归档（定调 + 待写死项）

- **质变 / 量变 / 「一次足够质变」**：尚无**操作定义**与**量变→阈值**分界；随**大量单词分析**推进后再写，以减少同类结构不一致。
- **形义优先 vs 音义**：**§2.1** 已写原则；**同一字母位**上形义与音变体义冲突时的**显式裁决顺序**仍缺——见 **§2.1（待补）**；当前优先厘清 **字母与音** 的对应，再系统性收口。
- **`a` 与 `e`（第二节「重过程」与第四节「量/变量/临界」）**：**不必在规则卡中单开章节强调**「过程 vs 结果谁为主轴」——那是给初学者的整体理念。要抓住的**实质**：**`a` 侧变化带动 `e` 侧变化**（例：全程 10、已走 3 为 `a` 量，同时 `e` 可表「剩 7」；`a` 到满程则 `e` 归零）；**`a` 足够可把 `e` 推到可质变态，须配坍缩手段才落定**。另一直觉例：施力（`a`）累积至表面破裂，**结果已可见**即 **`e` 落定 / 坍缩**。**`a`/`e` 是否要在拼写里突出**：**完全由该概念当前要聚焦什么决定**，无统一硬性要求。
- **`pa` 输出向**：与 **`ap`** 对举已有，**缺 §6 级范例锚点**；推演簿易停临时节点，见 **§2.2**。
- **`-mit` 族**：**submit / commit / transmit** 等与 **admit / permit** 是否**共用同一套 `mit` 抽象句**还是**逐词形参**，未定；**忌**泛化为「凡 `-mit` 皆同」，与 **ad vs per** 的精细分工冲突。细析随词例增加再补。
- **总判**：**`s`/`z`/`r` 蛇系**与 **`send` 系锁定**、**`h`/`r` 指向**等**多义并存**时的**择用/覆盖条件**，以及 **`vase` 英音**与器皿义之对读，多因**已分析单词仍偏少**，暂不单开硬规则；**当前工作向：按 §8 大量做单词分析**，稳定范例回写 **§6** / **§2**。
- **`h` 与 `r`（指向）**：**`h`** 已定 **朝向/方向向量**（**§6.21**）；**`r`** 已接**蛇系竖起**（**§3.2**、**§6.28**）；二者**细分工**仍**随词例再收**。
- **`advance` 内 `van` 块** vs **单词 `van`**：**同一 `v-a-n` 字母串**、**形参两套**（阶段推进 vs 凹空+壳内空间），见 **§6.23**、**§6.24**。
- **双写 `d` 通则**：**§4.3** 仅收 **`add`**；是否推广为「**同字母重复 = 操作迭代**」，**待定**。
- **`decide` 多米诺骨牌具象**：主推理见 **§6.6**（**分餐式决定**）；若将来要**单开**骨牌链式叙述，与 **§6.6** **并行**即可。
- **`si` / `is` / `st`≈`set`**：**§6.42** 已标明为**顺序对位 + 义理同脉**；**勿**当作所有含 `si`/`is`/`st` 的英语词的**统一词源公式**。
- **推进顺序（2026-03-28）**：**先夯实每个音 / 字母位的义位与边界**（逐条讲解、复述到满意），再系统回填 **§2** 与精选 **§6**。**§8** 单词流程仍以「例句 → 白话 → 切分」为准，但**宜在音义已共识的前提下使用**，以免底盘空转。

### 9.2) 词条与拆解（原列 + 延续）

- **`decide` / `cide`**：主链已定 → **§6.6**；**多米诺骨牌链式**仅作可选备忘（**§9.1**、**§6.6** 末注）。
- **`er` vs `or`**：历史与造词差异可能并存；**核心逻辑不受影响**，暂搁置，有灵感再回填。
- **`-ing` 音素拆解**（`i`/`n`/`g`）：延后。
- **`head` / `heading`**：已定标 → **§6.21**、**§6.25**；与 **`ending` 中 `d`**：**§4.1**、**§6.10**。
- **`sus`（`suspend` 等）**：前缀细拆**延后**。
- **`ina` / `ine`（块）**：**未定稿**。**§2**、**§2.7**、**§6.36** 等处写法为**工作假设**；**真正要收口的是 `n` 轴**（**收敛 / 内部 / 印迹**等与 **`na`/`ne` 布尔读法**如何分档、**`i` 侧**是否共用同一套叙事）——**词例够多后再集中重审**，**勿当死标**。
- **待复盘词（负向·零档·数档，与上条并案）**：**`minus`、`negative`、`nine`**——**先推进其他词**；积累到一定程度后**拉回与 `ina`/`ine`/`na`/`ne` 对读**。
